# スカウト分析（Google スプレッドシート連携）

送信した全スカウトについて、候補者プロフィールの分解（年齢・学歴・大学・現職・転職回数など）と
返信の有無を蓄積し、**週次・月次の返信率**を Google スプレッドシートに自動で図解・分析します。

## シート構成（自動生成・自動更新）

| シート | 内容 |
|---|---|
| **送信ログ** | 送信済み候補者1人=1行。プロフィール分解＋返信状態。**「返信あり」チェックボックスは手動入力可**（返信日・メモも保持される） |
| **週次サマリ** | 直近26週の送信数・返信数・返信率(%)・累計。**送信数×返信率のグラフ付き** |
| **月次サマリ** | 直近12ヶ月の同集計。グラフ付き |
| **セグメント分析** | 「いつ・誰に・どう送ると返信率が高いか」の11軸: 年齢帯別・学歴別・転職回数別・トーン別・送信枠別・会員クラス別・**曜日別・時間帯別・想定年収帯別・会員ステータス別・本文文字数帯別**。年齢帯・学歴・曜日・時間帯はグラフ付き |
| **傾向分析** | Claude が週1回自動生成する日本語の傾向分析（実績・トレンド・刺さっているセグメント・改善提案） |

- 返信率は「初回送信した週/月」に帰属します（例: 7/14送信→7/20返信 は 7/14の週の返信としてカウント）。
- 直近の週・月は返信がまだ届き得るため、返信率が低めに出ます。

## 返信の記録方法（2系統）

1. **自動検知**: 毎日の定期実行で2段構えで判定します。
   - **受信箱スキャン（主）**: メッセージ受信箱（inbox）に送信済み候補者からの
     メッセージがあるかを会員番号/mrccidで照合します（受信箱にある = 返信あり。
     辞退の返信も含む）。
   - **レジュメ再確認（副）**: 未返信の候補者のレジュメを再取得し、氏名の開示から
     判定します（2026-07-18 の偵察で実データ確認済み: 未返信は `candidateName=null`
     のまま。`hasContact` は「自社が接触済み」の意味のため判定に使いません）。
     対象はローテーションし、送信済み全員を数日で一巡します。
2. **手動チェック**: 送信ログの「返信あり」列にチェックを入れると、次回同期時にDBへ取り込まれます。
   自動検知より手動チェックが先でも問題ありません（両方の記録は統合され、返信=1が優先）。

※ シートは毎日全面的に書き直されますが、**「返信あり」「返信日」「メモ」列の手動入力は
会員番号をキーに引き継がれます**。他の列への手動編集は次回同期で消えます。

## 初回セットアップ（約10分・1回だけ）

### 1. Google Cloud でサービスアカウントを作成
1. https://console.cloud.google.com/ で新規プロジェクトを作成（例: `bizscout-analytics`）
2. 「APIとサービス」→「ライブラリ」→ **Google Sheets API** を検索して「有効にする」
3. 「APIとサービス」→「認証情報」→「認証情報を作成」→「サービスアカウント」
   - 名前: `bizscout`（任意）。ロールは不要（スキップ可）
4. 作成したサービスアカウント →「キー」タブ →「鍵を追加」→「新しい鍵を作成」→ **JSON** をダウンロード

### 2. スプレッドシートを作成して共有
1. https://sheets.new で空のスプレッドシートを作成（名前例: `スカウト分析`）
2. 「共有」で、サービスアカウントのメールアドレス（`bizscout@….iam.gserviceaccount.com`、
   鍵JSON内の `client_email`）を**編集者**として追加
3. URLの `https://docs.google.com/spreadsheets/d/`**この部分**`/edit` をコピー（スプレッドシートID）

### 3. GitHub に登録
リポジトリの **Settings → Secrets and variables → Actions** で:

| 種別 | 名前 | 値 |
|---|---|---|
| Secret | `GSHEET_SERVICE_ACCOUNT_B64` | **ダウンロードした鍵JSONの中身をそのまま全文貼り付け**（メモ帳で開いて全選択→コピー）。base64化した文字列でも可 |
| Variable | `BIZSCOUT_GSHEET_SPREADSHEET_ID` | 手順2-3のスプレッドシートID |

> 名前に「B64」とありますが、**base64化は不要**です（そのまま貼るのが確実。`{` で始まる
> JSONを自動判別します）。base64を使う場合: Mac `base64 -i 鍵.json | pbcopy` /
> Windows PowerShell `[Convert]::ToBase64String([IO.File]::ReadAllBytes("鍵.json")) | Set-Clipboard`

### 4. 動作確認
Actions タブ → scout → Run workflow → **mode=`analytics`** で実行。
成功するとスプレッドシートに5枚のシートとグラフが生成されます。
以降は毎日の定期実行後に自動で最新化されます。

## トラブルシューティング
- **シートが更新されない**: Actions のログで「Sync analytics」ステップを確認。
  `Google Sheets 未設定` と出る場合は手順3の Secret/Variable 名を再確認。
- **PERMISSION_DENIED**: スプレッドシートをサービスアカウントのメールに共有し忘れています（手順2-2）。
- **返信の自動検知が動かない/精度が低い**: mode=`probe-replies` を1回実行し、
  artifact の `reply_*` ファイルを開発者に共有してください（検知ロジックを実データで調整します）。

## ローカルでの確認（任意）
```bash
bizscout analytics report   # Sheets 設定なしで週次・月次・セグメントを端末表示
BIZSCOUT_GSHEET_SPREADSHEET_ID=... BIZSCOUT_GSHEET_CREDENTIALS=path/to/key.json \
  bizscout analytics sync   # 手元から同期
```
