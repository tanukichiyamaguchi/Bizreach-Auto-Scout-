# セレクタ設定ガイド

このガイドは、ビズリーチ画面の各ボタンや入力欄を、プログラムに正しく「どこか」を教えるための **セレクタ** を設定する手順を、なるべくやさしく説明します。専門的なコードを書く必要はありません。ブラウザの「開発者ツール」を使って画面の部品を1つずつ特定し、設定ファイルに書き写すだけです。

---

## 1. セレクタとは

プログラムは、ビズリーチの画面を人の代わりに操作します。そのとき「ログインボタン」「スカウト送信ボタン」などを、**セレクタ** という目印（住所のようなもの）で探します。

ビズリーチの画面は更新されることがあり、そのたびに目印が変わる可能性があります。そのため、**プログラム内蔵の初期値（想定値）をそのまま使うのではなく、実際の画面に合わせて設定しておく**ことが、安定運用のカギです。

設定の対象ファイルは次の2つです。

- `src/bizreach_scout/bizreach/selectors.py` … プログラム内蔵の**初期値（想定値）**。このファイルは**編集しません**。
- `config/bizreach_selectors.yaml` … あなたが**上書きするための設定ファイル**。ここに書いた値が優先されます。**空欄にした項目は初期値が使われます。**

---

## 2. 設定ファイルを用意する

まず雛形をコピーします。

```bash
cp config/bizreach_selectors.example.yaml config/bizreach_selectors.yaml
```

`config/bizreach_selectors.yaml` を開くと、項目名と空欄（`""`）が並んでいます。この空欄に、実際の値を記入していきます。

```yaml
# --- URL ---
login_url: ""
base_url: ""
search_url: ""

# --- ログイン ---
login_email: ""
login_password: ""
login_submit: ""
logged_in_marker: ""

# --- 検索結果 ---
result_link: ""
next_page: ""

# --- プロフィール ---
profile_root: ""
profile_member_no: ""

# --- スカウト送信 ---
scout_button: ""
scout_subject: ""
scout_body: ""
scout_send: ""
scout_confirm: ""
scout_sent_marker: ""
```

---

## 3. 各項目が何を指すか（一覧）

下表が、設定できる全項目と「画面のどこを指すか」「内蔵の初期値（想定値）」の一覧です。初期値は `src/bizreach_scout/bizreach/selectors.py` のものです。実画面と異なる場合は、`config/bizreach_selectors.yaml` で上書きしてください。

### URL

| 項目 | 指すもの | 内蔵の初期値（想定値） |
|------|----------|------------------------|
| `login_url` | ログイン画面のURL | `https://cr-support.jp/login` |
| `base_url` | サービスの基点URL（ログイン状態の判定で最初に開く） | `https://cr-support.jp` |
| `search_url` | 候補者検索のURL | `https://cr-support.jp/search` |

### ログイン

| 項目 | 指すもの | 内蔵の初期値（想定値） |
|------|----------|------------------------|
| `login_email` | メールアドレスの入力欄 | `input[name='email'], input[type='email']` |
| `login_password` | パスワードの入力欄 | `input[name='password'], input[type='password']` |
| `login_submit` | ログイン（送信）ボタン | `button[type='submit']` |
| `logged_in_marker` | ログイン成功の判定に使う要素（例: ログアウトリンク） | `text=ログアウト` |

### 検索結果

| 項目 | 指すもの | 内蔵の初期値（想定値） |
|------|----------|------------------------|
| `result_card` | 検索結果の候補者カード1件分 | `[data-testid='candidate-card'], .candidate-list-item` |
| `result_member_no` | カード内の会員番号表示 | `[data-testid='member-no'], .member-no` |
| `result_link` | 候補者のプロフィールへのリンク | `a[href*='/candidate/'], a[href*='/resume/']` |
| `next_page` | 検索結果の「次へ」 | `a[rel='next'], button[aria-label='次へ']` |

### プロフィール

| 項目 | 指すもの | 内蔵の初期値（想定値） |
|------|----------|------------------------|
| `profile_root` | プロフィール詳細の本文領域 | `main, [role='main'], .resume-detail` |
| `profile_member_no` | プロフィール上の会員番号 | `[data-testid='member-no'], .member-no` |

### スカウト送信

| 項目 | 指すもの | 内蔵の初期値（想定値） |
|------|----------|------------------------|
| `scout_button` | スカウト作成を開くボタン | `text=スカウト, text=プラチナスカウト, [data-testid='scout-button']` |
| `scout_subject` | 件名の入力欄 | `input[name='subject'], [data-testid='scout-subject']` |
| `scout_body` | 本文の入力欄 | `textarea[name='body'], [data-testid='scout-body']` |
| `scout_send` | 送信ボタン | `[data-testid='scout-send'], button:has-text('送信')` |
| `scout_confirm` | 送信確認ダイアログの「送信する / OK」ボタン | `button:has-text('送信する'), button:has-text('OK')` |
| `scout_sent_marker` | 送信完了の判定に使う表示（例: 「送信しました」） | `text=送信しました, text=スカウトを送信` |

> `config/bizreach_selectors.yaml` の雛形には主要項目だけが並んでいますが、`result_card` などの項目も同じ要領で1行追加すれば上書きできます。

## 4. 開発者ツールでの見つけ方（手順）

ここでは、目印（セレクタ）を実際の画面から見つける方法を説明します。例として Google Chrome を使います（他のブラウザでもほぼ同じです）。

1. ビズリーチに**手動でログイン**し、目的の画面（例: スカウト作成画面）を開きます。
2. 特定したい部品（例: 件名の入力欄）の上で**右クリック**し、メニューから **「検証」**（または「Inspect」）を選びます。
3. 画面の横や下に開発者ツールが開き、その部品にあたるHTMLの行がハイライトされます。
4. ハイライトされた行を見て、**目印として使える属性**を探します。優先順位は次のとおりです。
   - `data-testid="..."` があれば最優先（変わりにくい）。
   - 次に `id="..."` や `name="..."`。
   - ボタンなどは表示テキスト（例: 「送信する」）も使えます。

### 見つけた情報を「セレクタ」の書き方に直す

| 画面で見つけたもの（HTMLの属性） | YAML に書く値 |
|-------------------------------|---------------|
| `data-testid="scout-send"` | `[data-testid='scout-send']` |
| `id="subject"` | `#subject` |
| `name="body"` | `textarea[name='body']`（部品の種類に合わせる） |
| ボタンの表示テキスト「送信する」 | `button:has-text('送信する')` |
| ページ内に「送信しました」という文字が出る | `text=送信しました` |

> 上の `[...]` `#...` `text=...` といった書き方は、ブラウザ自動操作（Playwright）のセレクタ記法です。`,`（カンマ）で複数の候補を並べると、**どれか1つが見つかればOK**という意味になります（初期値もこの方式です）。画面の作りが変わっても当たりやすくなるので、迷ったら候補を複数並べておくと安全です。

---

## 5. YAML への書き方

見つけた値を、`config/bizreach_selectors.yaml` の該当項目に記入します。値は**引用符（`"`）で囲む**と安全です。

例（スカウト送信まわりを実画面に合わせて上書きする場合）:

```yaml
# --- スカウト送信 ---
scout_button: "[data-testid='scout-button']"
scout_subject: "#scout-subject"
scout_body: "textarea[name='body']"
scout_send: "button:has-text('スカウトを送信')"
scout_confirm: "button:has-text('送信する')"
scout_sent_marker: "text=送信が完了しました"
```

記入のポイント:

- **空欄（`""`）のままにした項目は、内蔵の初期値が使われます。** すべてを埋める必要はなく、初期値で問題ない項目はそのままで構いません。
- 値にカンマで複数候補を並べてよいです（例: `"#a, .b, [data-testid='c']"`）。
- 1か所書き換えたら、すぐに本番送信せず、必ず `BIZSCOUT_DRY_RUN=true` のドライランで動作を確かめてください（手順は **docs/運用手順.md** の「6. 初回ドライラン」）。

## 6. ログインがうまくいかないとき（2段階認証など）

ビズリーチで2段階認証（SMS や認証アプリ）が有効な場合、メールアドレスとパスワードだけでは自動ログインできません。その場合は、**最初に一度だけ手動でログインし、そのログイン状態を保存して再利用**します。

### 手順

1. `--no-headless` を付けて実行します。これでブラウザ画面が**目に見える状態で**起動します。

   ```bash
   bizscout run --source bizreach --search-url "https://cr-support.jp/search?saved=..." --no-headless
   ```

2. 立ち上がったブラウザで、**画面の指示に従って手動でログイン**します（2段階認証のコード入力もここで行います）。
3. ログインが完了すると、その**ログインセッションが自動的に保存**されます。保存先は `.env` の `BIZREACH_STORAGE_STATE`（既定 `data/sessions/bizreach_state.json`）です。
4. 以降は、このセッションが再利用されるため、毎回の2段階認証は不要になります（通常どおり `--no-headless` なしの実行でも動きます）。

### 補足

- セッションには有効期限があります。期限切れで再びログインを求められたら、上記の手順を**もう一度**実行して、セッションを更新してください。
- `logged_in_marker`（ログイン成功の目印）が実画面と合っていないと、ログイン済みでも「未ログイン」と誤判定されることがあります。うまく進まないときは、まず本ガイド「3〜5」の手順で `logged_in_marker` を実画面に合わせてください。

---

## 7. 設定後のチェック

セレクタを設定したら、本番の前に起動前チェックを実行します。

```bash
bizscout doctor
```

問題があれば指摘されます。指摘に従って `config/bizreach_selectors.yaml` を直し、すべて問題なしになってから、`BIZSCOUT_DRY_RUN=true` のドライランで実際の動作を確認してください。

それでも「ボタンが見つからない」などが起きる場合は、**docs/トラブルシューティング.md** の「セレクタ不一致」の項を参照してください。
